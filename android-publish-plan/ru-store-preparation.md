# План подготовки приложения к публикации в RuStore

## 1. Проверка требований RuStore

Перед публикацией необходимо убедиться, что приложение соответствует всем требованиям RuStore:

- [ ] Приложение должно быть адаптировано для российской аудитории
- [x] Поддержка русского языка (приложение уже на русском)
- [x] Отсутствие ссылок на запрещенные в России сервисы (файл google-services.json отсутствует, Google Mobile Services не используются)
- [ ] Соответствие законодательству РФ
- [x] Наличие политики конфиденциальности (файл создан: privacy-policy.md)
- [x] Поддержка 64-битных архитектур (Capacitor по умолчанию поддерживает)
- [ ] Поддержка Android 5.0+ (minSdkVersion 21+)

Проверка minSdkVersion:
```gradle
// android/variables.gradle
minSdkVersion = 24
```

Приложение требует Android 8.0 (API 24) и выше, что соответствует требованиям RuStore (Android 5.0+).

## 2. Настройка приложения для RuStore

### 2.1. Проверка идентификатора приложения

Текущий applicationId:
```gradle
applicationId "com.example.bylina"
```

Рекомендуется изменить на более подходящий для публикации. Предлагаю использовать `ru.slvctales.heroes` как более соответствующий названию приложения и домену проекта.

### 2.2. Обновление названия приложения

Текущее название в `capacitor.config.ts`:
```typescript
appName: 'Былина',
```

и в `strings.xml`:
```xml
<string name="app_name">Былина</string>
```

Название соответствует требованиям.

### 2.3. Подготовка иконок приложения

Доступные иконки в `public/`:
- `pwa-icon-192.png`
- `pwa-icon-512.png`
- `splash/splash.png`

Для RuStore необходимы:
- Иконка приложения (512x512 PNG)
- Сплэш-экран (рекомендуется)

Использовать `public/pwa-icon-512.png` как основную иконку.

### 2.4. Добавление политики конфиденциальности

Необходимо создать файл политики конфиденциальности и добавить ссылку в приложение.

## 3. Сборка APK/AAB

### 3.1. Подготовка к сборке

Убедиться, что:

- Настроена подпись приложения через environment variables
- Установлены все зависимости
- Приложение корректно работает в режиме release

### 3.2. Сборка приложения

```bash
# Сборка через Gradle
cd android
./gradlew.bat assembleRelease
```

Для сборки AAB (рекомендуется для RuStore):

```bash
# Сборка Android App Bundle
cd android
./gradlew.bat bundleRelease
```

Результат будет находиться в:
- APK: `android/app/build/outputs/apk/release/app-release.apk`
- AAB: `android/app/build/outputs/bundle/release/app-release.aab`

## 4. Подготовка материалов для публикации

Необходимо подготовить следующие материалы:

### 4.1. Скриншоты приложения

Сделать не менее 5 скриншотов в разрешении 1080x1920 или 1440x2560.

### 4.2. Видео-демонстрация (опционально)

Подготовить короткое видео (до 30 секунд) с демонстрацией функционала.

### 4.3. Описание приложения

Подготовить:
- Краткое описание (до 80 символов)
- Полное описание (до 4000 символов)
- Ключевые слова (теги)

Пример краткого описания:
"Эпическая RPG по мотивам славянских сказаний"

### 4.4. Контактная информация

Указать:
- Email для связи
- Сайт (если есть)
- Ссылка на политику конфиденциальности

## 5. Процесс публикации в RuStore

### 5.1. Регистрация разработчика

Зарегистрироваться на https://dev.rustore.ru/ как разработчик.

### 5.2. Создание нового приложения

В панели разработчика:
- Нажать "Добавить приложение"
- Заполнить основную информацию
- Загрузить иконку приложения

### 5.3. Загрузка сборки

- Загрузить AAB или APK файл
- Указать версию приложения
- Выбрать целевые устройства

### 5.4. Заполнение описания

- Добавить краткое и полное описание
- Загрузить скриншоты и видео
- Указать категорию приложения

### 5.5. Настройка контента

- Указать возрастной рейтинг
- Отметить тип приложения (игра)
- Указать наличие покупок в приложении (если есть)

### 5.6. Отправка на модерацию

Проверить все данные и отправить приложение на модерацию.

## 6. Пост-публикационные действия

- Мониторинг отзывов пользователей
- Регулярное обновление приложения
- Анализ статистики установок
- Реагирование на замечания модераторов при необходимости

## Примечания

1. Приложение использует Capacitor, что уже настроено для Android сборки
2. Подпись приложения настроена через environment variables, что безопасно для CI/CD
3. Приложение уже имеет русский интерфейс, что соответствует требованиям RuStore
4. Рекомендуется использовать AAB вместо APK для лучшего управления размером приложения
5. Google Mobile Services не используются (отсутствует google-services.json), что обеспечивает независимость от Google
6. Политика конфиденциальности создана и готова к публикации
7. Идентификатор приложения рекомендуется изменить перед публикацией
