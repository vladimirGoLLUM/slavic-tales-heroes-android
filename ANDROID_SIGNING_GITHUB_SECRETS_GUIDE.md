# Руководство по добавлению ключей подписи APK в GitHub Secrets

## Что такое ключи подписи APK?

Ключи подписи необходимы для того, чтобы:
- Подтвердить, что вы являетесь разработчиком приложения
- Обеспечить безопасность обновлений (новые версии могут устанавливаться поверх старых только при совпадении подписи)
- Опубликовать приложение в Google Play

**ВАЖНО**: Эти ключи являются конфиденциальными! Никогда не коммитьте их в репозиторий.

## Генерация keystore (если еще не создан)

Если у вас еще нет файла ключей, создайте его с помощью команды:

```bash
keytool -genkey -v -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
```

Ответьте на вопросы:
- Пароль для keystore
- Имя, фамилия
- Название организации
- Город, область
- Двухбуквенный код страны

Файл `my-release-key.keystore` нужно сохранить в безопасном месте.

## Извлечение параметров из keystore

1. **Пароль keystore** - тот, который вы задали при создании
2. **Пароль ключа** - обычно совпадает с паролем keystore
3. **Алиас ключа** - имя, которое вы задали (в примере выше `my-key-alias`)

## Преобразование keystore в base64

Для GitHub Secrets нужно преобразовать keystore файл в base64:

```bash
# На Windows (PowerShell)
[Convert]::ToBase64String([IO.File]::ReadAllBytes("path/to/my-release-key.keystore")) | Out-File "keystore-base64.txt" -Encoding utf8

# На macOS/Linux
base64 < path/to/my-release-key.keystore > keystore-base64.txt
```

Скопируйте содержимое файла `keystore-base64.txt` - это будет значение для секрета.

## Необходимые GitHub Secrets

В настройках репозитория GitHub нужно создать следующие секреты:

| Имя секрета | Описание | Источник |
|-------------|----------|----------|
| `RELEASE_STORE_FILE` | Содержимое keystore в base64 | Результат команды base64 |
| `RELEASE_STORE_PASSWORD` | Пароль от keystore | Ваш пароль |
| `RELEASE_KEY_ALIAS` | Алиас ключа | Имя, заданное при создании |
| `RELEASE_KEY_PASSWORD` | Пароль ключа | Ваш пароль (часто совпадает с RELEASE_STORE_PASSWORD) |

## Пошаговая инструкция

1. Перейдите в репозиторий на GitHub
2. Нажмите `Settings` → `Secrets and variables` → `Actions`
3. Нажмите `New repository secret`
4. Введите имя секрета (например, `RELEASE_STORE_FILE`)
5. В поле значение вставьте соответствующие данные
6. Нажмите `Add secret`
7. Повторите для всех четырех секретов

## Рекомендации по безопасности

- Храните оригинальный keystore файл в надежном месте (например, на зашифрованном USB-накопителе)
- Никогда не коммитьте keystore файл или его base64-представление в репозиторий
- Ограничьте доступ к GitHub Secrets только доверенным разработчикам
- Регулярно меняйте пароли (если возможно)
- Сделайте резервную копию keystore - если вы его потеряете, не сможете обновлять приложение в Google Play!
