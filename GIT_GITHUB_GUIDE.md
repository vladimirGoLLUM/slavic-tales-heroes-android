# Руководство по работе с Git и GitHub в Android Studio

Это пошаговое руководство поможет вам эффективно использовать систему контроля версий Git и интеграцию с GitHub прямо из Android Studio.

## 1. Настройка Git в Android Studio

### Установка Git
1. Скачайте и установите Git с официального сайта: https://git-scm.com/
2. Во время установки выберите опцию "Add Git to PATH"
3. Перезапустите Android Studio

### Настройка учетных данных
1. Откройте Android Studio
2. Перейдите в `File` → `Settings` (или `Ctrl+Alt+S`)
3. В левом меню выберите `Version Control` → `Git`
4. Убедитесь, что путь к Git указан правильно (обычно `git` или `C:\Program Files\Git\bin\git.exe`)
5. Перейдите в `Version Control` → `GitHub`
6. Нажмите `Add account` и войдите в свой аккаунт GitHub

## 2. Подключение к существующему репозиторию

### Клонирование репозитория
1. Закройте текущий проект (если открыт)
2. На стартовом экране выберите `Get from VCS`
3. В поле URL вставьте адрес вашего репозитория GitHub (например, `https://github.com/username/repository.git`)
4. Выберите директорию для проекта
5. Нажмите `Clone`

### Подключение к уже существующему проекту
1. Откройте проект в Android Studio
2. Перейдите в `VCS` → `Import into Version Control` → `Create Git Repository`
3. Выберите корневую директорию проекта
4. Перейдите в `VCS` → `Git` → `Remotes`
5. Нажмите `+` и добавьте удаленный репозиторий с URL вашего GitHub

## 3. Основные операции с Git

### Pull (получение обновлений)
1. Перейдите в `VCS` → `Git` → `Pull`
2. Или используйте горячие клавиши `Ctrl+T`
3. Выберите ветку (обычно `origin/main`)
4. Нажмите `Pull`

**Совет**: всегда выполняйте Pull перед началом работы, чтобы получить последние изменения.

### Commit (фиксация изменений)
1. Перейдите в `VCS` → `Commit`
2. Или используйте горячие клавиши `Ctrl+K`
3. В левой панели вы увидите все измененные файлы
4. Отметьте файлы, которые хотите включить в коммит
5. Введите осмысленное сообщение коммита (например, "Исправлено отображение полноэкранного режима")
6. Нажмите `Commit and Push` или просто `Commit`

### Push (отправка изменений)
1. Перейдите в `VCS` → `Git` → `Push`
2. Или используйте горячие клавиши `Ctrl+Shift+K`
3. Выберите коммиты для отправки
4. Нажмите `Push`

**Важно**: Push отправляет ваши коммиты в удаленный репозиторий на GitHub.

## 4. Работа с ветками

### Создание новой ветки
1. Перейдите в `Git` → `Branches` (или нажмите `Alt+` в правом нижнем углу)
2. Выберите `New Branch`
3. Введите имя ветки (например, `feature/fullscreen-fix`)
4. Нажмите `Create`

### Переключение между ветками
1. Нажмите на имя текущей ветки в правом нижнем углу
2. Или перейдите в `Git` → `Branches`
3. Выберите нужную ветку из списка
4. Нажмите `Checkout`

### Слияние веток
1. Переключитесь на ветку, в которую хотите влить изменения (обычно `main`)
2. Перейдите в `Git` → `Merge Changes`
3. Выберите ветку для слияния
4. Нажмите `Merge`

## 5. Разрешение конфликтов слияния

Конфликты возникают, когда один и тот же файл изменен в разных ветках.

### Обнаружение конфликтов
1. При попытке слияния или pull, Android Studio покажет окно с конфликтами
2. Файлы с конфликтами будут отмечены красным

### Разрешение конфликта
1. Откройте файл с конфликтом
2. Вы увидите блоки кода в формате:
```
<<<<<<< HEAD
Ваш код
=======
Код из удаленной ветки
>>>>>>> branch-name
```
3. Решите, какой код оставить, или объедините изменения
4. Удалите маркеры конфликта (`<<<<<<<`, `=======`, `>>>>>>>`)
5. Сохраните файл
6. Перейдите в `Git` → `Commit`
7. Отметьте файл как решенный и сделайте коммит

## 6. Просмотр истории изменений

### История коммитов
1. Перейдите в `VCS` → `Git` → `Show Git Log`
2. Или нажмите правой кнопкой на проекте → `Git` → `Show History`
3. Вы увидите список всех коммитов с авторами и сообщениями

### Просмотр изменений в файле
1. Откройте файл
2. Щелкните правой кнопкой → `Git` → `Show History`
3. Или используйте `Alt+9` для открытия панели `Version Control`
4. Перейдите на вкладку `Log`

## 7. Интеграция с GitHub

### Работа с Issues
1. Перейдите в `View` → `Tool Windows` → `GitHub`
2. В панели GitHub вы можете:
   - Просматривать открытые issues
   - Создавать новые issues
   - Привязывать коммиты к issues

### Pull Requests
1. Для создания Pull Request:
   - Сделайте коммиты в своей ветке
   - Нажмите `Ctrl+Shift+K` для Push
   - Android Studio предложит создать Pull Request
   - Или создайте его вручную на GitHub

### Просмотр Pull Requests
1. Перейдите в `View` → `Tool Windows` → `Pull Requests`
2. Вы можете просматривать, комментировать и проверять код в PR

## Полезные советы

1. **Регулярно делайте коммиты** - небольшие, осмысленные изменения
2. **Пишите понятные сообщения коммитов** - что было сделано и зачем
3. **Всегда делайте Pull перед началом работы** - чтобы избежать конфликтов
4. **Используйте ветки для новых функций** - не работайте напрямую в main
5. **Проверяйте изменения перед коммитом** - убедитесь, что коммитите только нужные файлы

Теперь вы готовы эффективно работать с Git и GitHub в Android Studio!
